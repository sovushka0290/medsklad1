# МедСклад — Система автоматизации складского учёта стоматологической клиники

**МедСклад** — это мобильное решение (React Native / Expo) и защищенный бэкенд (Node.js / Express / Prisma), разработанное специально для Областной детской стоматологической клиники. Система позволяет автоматизировать учет медикаментов, расходных материалов и отслеживать дефицитные позиции в реальном времени.

---

## 🏗️ Архитектура системы

Система построена по трехзвенной клиент-серверной архитектуре:

1. **База данных (Supabase PostgreSQL):** Хранит номенклатуру препаратов, остатки по партиям на конкретных складах/кабинетах, а также лог всех транзакций аудита.
2. **Серверная часть (Express + TypeScript + Prisma):** Обрабатывает бизнес-логику транзакций (приход/списание), осуществляет быстрый поиск, управляет остатками и защищает запись данных через авторизационный middleware.
3. **Мобильный клиент (React Native / Expo / NativeWind):** Интерфейс для медсестер и кладовщиков, поддерживающий сканирование штрихкодов с камеры, списание/оприходование препаратов в один тап, просмотр дефицита и привязку штрихкодов к товарам прямо со сканера.

---

## 🔒 Безопасность и Оптимизация

* **Авторизация API:** Все изменяющие базу маршруты (создание транзакций, привязка штрихкодов) закрыты с помощью `authMiddleware` и требуют валидный `x-api-key` в заголовках запроса.
* **Производительность бэкенда:** Базовый API получения лекарств (`GET /api/medications`) оптимизирован с использованием пагинации (поддержка query-параметров `page` и `limit`).
* **Оптимизация сети (Mobile):** Внедрена подготовка и сжатие снимков на стороне мобильного приложения (до 1024px ширины, качество 0.6) через `expo-image-manipulator` перед отправкой файлов для экономии трафика и ускорения работы.

## 🛠️ Спецификация API (Основные эндпоинты)

### Медикаменты и Склад

* **`GET /api/medications`**
  * *Назначение:* Возвращает список всех товаров с остатками по партиям.
  * *Параметры:* `barcode` (поиск по штрихкоду), `page`, `limit` (пагинация).
* **`GET /api/medications/search?q=запрос`**
  * *Назначение:* Быстрый поиск номенклатуры по имени или штрихкоду (регистронезависимый).
* **`GET /api/medications/critical`**
  * *Назначение:* Фильтрация дефицитных товаров (остаток которых меньше лимита `minQuantity`).
* **`PUT /api/medications/:id`**
  * *Назначение:* Привязать отсканированный штрихкод к существующей карточке товара.
  * *Защита:* Требует заголовок `x-api-key`.

### Складские Локации
* **`GET /api/locations`**
  * *Назначение:* Получение справочника всех кабинетов и складов клиники.

### Операции (Транзакции)
* **`POST /api/transactions`**
  * *Назначение:* Проведение складской операции (приход/расход).
  * *Защита:* Требует заголовок `x-api-key`.
  * *Тело запроса:*
    ```json
    {
      "type": "INCOME", // или "OUTFLOW" для списания
      "quantity": 1,
      "medicationId": 5,
      "locationId": 2
    }
    ```
* **`GET /api/transactions`**
  * *Назначение:* Журнал аудита складских движений (история).

## 🚀 Быстрый запуск

### Шаг 1: Настройка базы данных и бэкенда

1. Перейдите в папку бэкенда:
   ```bash
   cd backend
   ```
2. Создайте файл `.env` на основе примера и укажите URL вашей базы данных PostgreSQL:
   ```env
   DATABASE_URL="postgresql://username:password@host:6543/postgres?pgbouncer=true"
   DIRECT_URL="postgresql://username:password@host:5432/postgres"
   API_KEY="MedSkladSecretKey123"
   ```
3. Установите зависимости:
   ```bash
   npm install
   ```
4. Синхронизируйте схему и наполните базу демонстрационными стоматологическими материалами:
   ```bash
   npx prisma db push
   npx prisma db seed
   ```
5. Запустите Express-сервер в режиме разработки:
   ```bash
   npm run dev
   ```

### Шаг 2: Запуск мобильного приложения (Expo)

1. Перейдите в папку мобильного клиента:
   ```bash
   cd ../mobile
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Запустите Expo-сервер разработки:
   ```bash
   npm run start
   ```
4. Откройте приложение в эмуляторе (нажав `a` для Android или `i` для iOS) или отсканируйте QR-код через приложение **Expo Go** на вашем смартфоне.

## 📝 Описание Демо-данных (Seeding)

База данных инициализируется **21 стоматологическим препаратом** (Ультракаин, Филтек, Эстелайт, Адгезор, нитриловые перчатки и др.), распределенными по **4 локациям**:
* Главный склад
* Кабинет №1 (Терапия)
* Кабинет №2 (Хирургия)
* Кабинет №3 (Ортодонтия)

Некоторые материалы (например, пломбировочный материал *Филтек Z250* и *нитриловые перчатки*) искусственно засидированы с низким остатком, чтобы при открытии приложения на вкладке **«Дефицит»** мгновенно отображались предупреждения о критических остатках.
